# Secrets Management (sops-nix)

Secrets are encrypted with [sops](https://github.com/getsops/sops) using [age](https://github.com/FiloSottile/age) keys and decrypted at activation time by [sops-nix](https://github.com/Mic92/sops-nix).

## How it works

Each host decrypts secrets using its **SSH host key** — no separate age key file is needed. sops-nix derives an age key from the SSH host key automatically at activation time.

- **NixOS-level** (`sops.age.sshKeyPaths = [ "/etc/ssh/ssh_host_ed25519_key" ]`): runs as root on every boot, places system secrets into `/run/secrets/`
- **Home-manager-level** (`sops.age.sshKeyPaths = [ "~/.ssh/id_ed25519" ]`): runs as the user on login, places user secrets

The age public keys in `.sops.yaml` are derived from each machine's SSH host key via `ssh-to-age`.

### Asymmetric encryption — why you can encrypt for all hosts from one machine

age uses **asymmetric encryption**:

- **Public key** (`age1...`) — safe to share, stored in `.sops.yaml`. Used to *encrypt*.
- **Private key** — derived from the SSH host key at runtime. Used to *decrypt*.

When sops creates or edits a file, it generates a one-time random data key, encrypts the file contents with it, then encrypts that data key once per recipient using each recipient's public key. Each host independently decrypts using its own SSH host key — no host can read another's private key.

## Files

| File | Recipients | Contents |
|---|---|---|
| `common.yaml` | thinkpad, personal, workstation | Shared system secrets (GitHub token, etc.) |
| `deus.yaml` | thinkpad, personal, workstation | SSH user key for `deus` |
| `soumya.yaml` | thinkpad, workstation | SSH user key + password for `soumya` |
| `rclone.yaml` | rclone, personal, thinkpad | rclone configuration |
| `../.sops.yaml` | Repo root | Age key recipients and creation rules |

## Editing secrets

```bash
sops secrets/deus.yaml    # decrypts, opens $EDITOR, re-encrypts on save

# If sops isn't in PATH:
nix shell nixpkgs#sops -c sops secrets/deus.yaml
```

## Adding a new host

1. Install NixOS and boot — the SSH host key is auto-generated by `sshd-keygen.service` on first boot
2. Get the age public key: `cat /etc/ssh/ssh_host_ed25519_key.pub | ssh-to-age`
3. Add it to `.sops.yaml` with a new YAML anchor
4. Re-encrypt any shared files so the new host can read them:
   ```bash
   sops updatekeys secrets/common.yaml
   sops updatekeys secrets/deus.yaml
   ```
5. `just install` on the new host — no age key bootstrap required

## Reinstalling a host

No manual key restoration needed. On first boot after install:
1. NixOS auto-generates `/etc/ssh/ssh_host_ed25519_key`
2. sops-nix derives the age key from it and decrypts all secrets automatically
3. The SSH host key changes after reinstall — update `known_hosts` on other machines

## Re-encrypting after adding a recipient

```bash
# After adding a new age key to .sops.yaml:
sops updatekeys secrets/common.yaml
```

## User SSH keys

`ssh.nix` places a user's SSH private key at `~/.ssh/id_ed25519` via NixOS-level sops if `secrets/<username>.yaml` exists. The placement happens on every boot (system activation), not just on login.

Secret name format: `ssh-<username>` (e.g., `ssh-soumya`, `ssh-deus`) — unique per user to avoid namespace collisions on multi-user hosts.
