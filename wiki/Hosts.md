# Adding a new host

## 1. Create the files

```bash
mkdir -p modules/hosts/newhost
```

Files needed:
- `modules/hosts/newhost/default.nix` — tracked by git, contains the host declaration
- `modules/hosts/newhost/_hardware-configuration.nix` — generated by `nixos-generate-config`, `_`-prefixed so import-tree ignores it
- `modules/hosts/newhost/_disko-config.nix` — disk layout

## 2. Write `default.nix`

```nix
{ den, ... }:
{
  den.hosts.x86_64-linux.newhost = {
    home-manager.enable = true;
    wm = "swayfx";
    greetdUser = "deus";
    greetdSessionBin = "sway";
    nhCleanEnabled = true;
    isLaptop = false;
    nvimLanguages = [ "lua" "nix" "python" ];
    users.deus = { };
  };

  den.aspects.newhost = {
    nixos = { ... }: {
      imports = [ ./_hardware-configuration.nix ./_disko-config.nix ];
      time.timeZone = "Asia/Kolkata";
      i18n.defaultLocale = "en_US.UTF-8";
    };

    includes = [
      # Core system bundles
      den.aspects.base-system        # nix-config, networking, boot, sudo, disko, nix-tools, zram
      den.aspects.graphical-session  # pipewire, graphics, greetd, dms, fonts, gnome-keyring
      den.aspects.desktop-services   # flatpak, appimage, WebKit/GTK
      # Networking / VPN
      den.aspects.tailscale
      # Add host-specific aspects below (avahi, mullvad, lact, laptop, bluetooth, ...)
    ];
  };
}
```

## 3. Install NixOS and boot

NixOS auto-generates the SSH host key at `/etc/ssh/ssh_host_ed25519_key` on first boot via `sshd-keygen.service`. No manual key generation needed.

## 4. Get the age public key and register it in `.sops.yaml`

On the new machine after first boot:

```bash
cat /etc/ssh/ssh_host_ed25519_key.pub | ssh-to-age
# → age1xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

Add it to `.sops.yaml` on workstation:

```yaml
keys:
  - &thinkpad age1...
  - &personal age1...
  - &workstation age1...
  - &newhost age1...    # paste the output from above

creation_rules:
  - path_regex: secrets/deus\.yaml$
    key_groups:
      - age:
          - *thinkpad
          - *personal
          - *workstation
          - *newhost    # add this line
```

Re-encrypt shared files so the new host can decrypt them:

```bash
sops updatekeys secrets/deus.yaml
sops updatekeys secrets/common.yaml
```

## 5. Stage and build

```bash
git add modules/hosts/newhost/
just install   # on the new host — no age key bootstrap needed
```

## 6. Enable rclone (optional, deus only)

Add the new host to rclone.yaml's recipients in `.sops.yaml`:

```yaml
- path_regex: secrets/rclone\.yaml$
  key_groups:
    - age:
        - *rclone
        - *personal
        - *thinkpad
        - *newhost
```

```bash
sops updatekeys secrets/rclone.yaml
```

Then add `"rclone"` to `extraAspects` in the host declaration.
